# Моя кухня

Личное веб-приложение для управления рецептами, планом питания и списком покупок. Без бэкенда, без сборщиков, без аккаунтов — чистый HTML + CSS + Vanilla JS.

**Три ключевые фичи:**
- ✅ Галочки на покупках (с прогресс-баром, сохраняются между сессиями)
- 🔢 Пересчёт количества ингредиентов под нужное число порций
- 💬 Заметки после приготовления (для каждого рецепта отдельно)

Все пользовательские данные (галочки, порции, заметки) хранятся в `localStorage` браузера.

## Структура

```
.
├── index.html          ← разметка + точка входа
├── style.css           ← стили (тёплая палитра, мобайл-фёрст)
├── app.js              ← вся логика (роутинг, рендер, localStorage)
├── data.json           ← извлечённые данные (рецепты, план, покупки, профиль)
├── source-data/        ← исходные md-файлы — единый источник правды
│   ├── my-food-profile.md
│   ├── my-recipes.md
│   ├── meal-plan-1.md
│   └── shopping-1.md
└── scripts/
    └── build.js        ← генератор data.json (Node.js)
```

## Локальный запуск

Из-за `fetch('data.json')` приложение требует HTTP-сервера — двойной клик по `index.html` не сработает (Chrome заблокирует CORS). Запусти простой сервер:

```bash
# В папке проекта:
python3 -m http.server 8000
# или:
npx http-server -p 8000
```

Открой [http://localhost:8000](http://localhost:8000).

## Деплой на GitHub Pages

1. Создай публичный репозиторий на GitHub: [github.com/new](https://github.com/new). Не ставь галочки «Add README» / «Add .gitignore».
2. Подключи репо и запушь:
   ```bash
   git remote add origin https://github.com/ТВОЙ_НИК/ИМЯ_РЕПО.git
   git branch -M main
   git push -u origin main
   ```
3. В репозитории: **Settings → Pages → Source: Deploy from a branch → main / (root) → Save**.
4. Подожди 1–2 минуты — сайт будет на `https://ТВОЙ_НИК.github.io/ИМЯ_РЕПО/`.

На телефоне: открой ссылку в Safari/Chrome → «Поделиться» → «На экран Домой».

## Обновление данных

Когда приходит новый план или меняется список покупок:

1. Замени соответствующий md-файл в `source-data/`.
2. Перегенерируй `data.json`:
   ```bash
   node scripts/build.js
   ```
3. Запушь изменения:
   ```bash
   git add data.json source-data/
   git commit -m "обновил план"
   git push
   ```
4. Через ~1 минуту сайт обновится.

Локальные данные (галочки, порции, заметки) **не пострадают** — они в `localStorage`, не в файлах сайта.

## Добавление новых рецептов

Рецепты лежат в `scripts/build.js` как массив `recipes`. Каждый — объект с полями:
- `id` — стабильный slug (используется в URL и localStorage)
- `title`, `emoji`, `cat` (id категории), `time` (минуты), `servings` (базовое число порций)
- `serving` — строка «с чем подавать»
- `groups` — массив групп ингредиентов: `[{title, items: [{name, qty, unit, note?}]}]`
- `steps` — массив шагов: `[{title, minutes, body}]`
- `hacks` — массив строк с лайфхаками
- `storage` — строка про хранение (опционально)

После правки запусти `node scripts/build.js` — `data.json` пересоберётся.

## Сброс данных в браузере

В приложении: **Профиль → Сбросить все данные**. Удалит галочки, выбранные порции и заметки. Сами рецепты не пострадают.

## Что НЕ умеет (намеренно)

- Тёмная тема, таймеры на шагах, чекбоксы на шагах — пропущено сознательно.
- Нет поиска (рецептов мало).
- Нет авторизации, мультипользователя, синхронизации — это персональный сайт на одного.
